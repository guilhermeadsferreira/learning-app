# PEDAGOGY.md — Metodologia de Ensino

## 1. Objetivo

Este documento define a metodologia pedagógica da plataforma **Learning Engine**.

A plataforma foi projetada para ensinar tópicos técnicos através de:

* microlearning
* prática ativa
* progressão gradual
* revisão estruturada
* feedback imediato

A metodologia busca maximizar retenção, motivação e confiança do aluno.

---

# 2. Metodologia de Aprendizado

A plataforma combina quatro abordagens principais:

## Active Learning

O aluno aprende fazendo.

Cada conceito deve ser acompanhado de um exercício prático.

---

## Mastery Learning

O aluno deve demonstrar entendimento antes de avançar.

Isso é feito através de:

* exercícios
* desafios
* mini projetos

---

## Scaffolding

A dificuldade aumenta gradualmente.

Estrutura típica:

1. explicação guiada
2. exemplo
3. exercício assistido
4. exercício independente
5. desafio

---

## Retrieval Practice

O aluno deve recuperar informações da memória.

Isso é implementado com:

* quizzes
* desafios
* revisões posteriores

---

# 3. Estrutura obrigatória de lições

Toda lição deve seguir o seguinte formato:

1. Hook
2. Analogia
3. Explicação
4. Exemplo
5. Desafio
6. Feedback
7. Lembre-se

---

## 3.1 Hook

Introduz o problema ou pergunta central da lição.

Exemplo:

> Como React sabe quando atualizar a interface?

---

## 3.2 Analogia

Uma analogia concreta ajuda a conectar o conceito a algo familiar.

Exemplo:

> Imagine um post-it colado na tela que guarda um valor.

---

## 3.3 Explicação

Explicação curta e objetiva.

Regras:

* máximo 6 linhas
* evitar parágrafos longos
* focar na ideia central

---

## 3.4 Exemplo

Código completo demonstrando o conceito.

---

## 3.5 Desafio

Toda lição deve ter ao menos um exercício.

Tipos de exercício:

* completar código
* corrigir bug
* implementar componente
* responder quiz
* debugging

---

## 3.6 Feedback

Explicação da solução.

Deve incluir:

* por que funciona
* erro comum
* boas práticas

---

## 3.7 Lembre-se

Resumo curto do conceito.

Exemplo:

> useState retorna dois valores: o estado e a função que atualiza esse estado.

---

# 4. Estrutura de módulos

Cada módulo segue a seguinte progressão:

1. Introdução
2. Conceitos
3. Exercícios guiados
4. Desafios
5. Mini projeto
6. Revisão

---

# 5. Progressão de dificuldade

Desafios devem seguir progressão clara:

Nível 1 — completar código
Nível 2 — editar código
Nível 3 — criar código
Nível 4 — resolver problema

---

# 6. Tipos de exercício suportados

## Code Completion

Aluno completa um trecho faltante.

---

## Code Correction

Aluno corrige um erro existente.

---

## Implementation

Aluno cria código do zero.

---

## Quiz

Perguntas conceituais.

---

## Debugging

Aluno precisa identificar um erro.

---

# 7. Revisão

O sistema deve permitir revisão de conteúdos.

Tipos:

* revisão conceitual
* revisão prática
* repetição de desafios

---

# 8. Mini projetos

Cada módulo deve terminar com um pequeno projeto.

Exemplos:

* contador
* lista de itens
* formulário simples
* componente reutilizável

---

# 9. Regras pedagógicas

Regras obrigatórias:

* cada lição deve ter desafio
* explicações devem ser curtas
* analogias devem ser concretas
* cada conceito deve ter exemplo
* cada módulo deve ter mini projeto

---

# 10. Objetivo pedagógico

O objetivo final é que o aluno:

* entenda conceitos fundamentais
* saiba aplicá-los
* desenvolva confiança
* consiga construir pequenas aplicações
